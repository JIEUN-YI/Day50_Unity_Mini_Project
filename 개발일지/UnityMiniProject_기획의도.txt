Day 50_Unity_Mini_Project

<1차 교수님 코멘트>
점프/슬라이딩 이동
배경이 플레이어에게 다가오는 식으로 작업해야함

게임이 길어질수록 프레임 드랍 조심
- 차근차근 오브젝트/배경 생성
- 스크롤링 기법 적용하여 해결 (콘텐츠 추가 시 사용)
==================================================

제작목표 : 쿠키런을 분석하여 제작한 런닝게임
장르 : 런닝게임

제작 최종 목표
1. 플레이어와 맵을 제작하여 플레이어의 이동을 완벽하게 구현할 것
2. 맵의 장애물을 구현할 것
3. 게임의 진행을 구현할 것
4. 아이템을 사용할 수 있게 구현할 것
5. 가능하다면 상태패턴과 싱글톤 패턴을 사용할 것

구현 내용
1. 맵
- 플레이어의 체력이 0이상인 동안 발판이 무한으로 생성
- 발판이 생성되는 동안 장애물 3종이 랜덤하게 출력
   : 아래에서 올라오는 장애물
      - 2단 점프로 피할 수 있는 높이까지 총 2종이 생성
      - 플레이어 점프로 피할 수 있음
   : 위에서 내려오는 장애물
      - 플레이어가 슬라이드를 하는 동안 피할 수 있음

2. 플레이어
- 플레이어의 체력은 게임진행 중 자연스럽게 감소한
   : 슬라이더 UI로 체력의 감소가 눈에 보이게 구현
- 플레이어가 장애물과 부딪힌 경우 체력이 일정량 감소
   : 플레이어가 아파하는 or 깜빡이는 애니메이션을 구현
- 플레이어는 장애물을 점프하여 피하거나 슬라이드 하여 피함
- 체력이 0이 되는 경우 게임이 종료

3. 아이템
- 게임이 진행되는 동안 랜덤으로 아이템이 생성
- 기본아이템
   : 게임이 진행되는 동안 점프와 슬라이드를 사용하여 젬을 먹어 점수를 획득
   : 젬은 맵의 장애물을 피한 자리에 생성
- 특수아이템
   : 아이템을 먹으면 일정시간동안 특수한 능력 활성화
      - 먹으면 거대해지는 아이템
         : 일정시간동안 장애물의 방해를 무력화
      - 먹으면 젬을 빨아들이는 아이템
         : 일정시간동안 젬을 자동으로 흡수

4. 게임의 진행
- 플레이어의 체력이 0이 될 때까지 무한히 발판을 만들어 게임을 진행
- 플레이어의 체력이 0이되면 게임을 종료하고 최종 점수를 화면에 출력
- 최고점수를 저장하고 게임을 다시 시작

작업 목록

1. 기본 기능의 구현
- 맵 [2024.09.25] 1차 기능 구현 완료 
   : Ground / Top 배경
      - Ground와 Top은 일정하게 고정
      - 장애물(과 추후 배경의)이동으로 맵이 이동하는 효과를 구현
   : 3종이 랜덤하게 출력되는 장애물(Hurdle)
      - 각 장애물에 Object Pool을 적용하여 대여/반납하여 사용
      - 장애물은 화면 밖에서 생성되어 점차 플레이어쪽으로 움직이며 화면 밖으로 나가며 삭제
- 플레이어 [2024.09.25] 1차 기능 구현 완료
   : 시간이 지날수록 줄어드는 체력
   : 2단 점프와 슬라이드 기능
   : 장애물과 충돌 시 체력의 감소
- 게임
   : 아무 키나 누르면 게임의 실행
   : 플레이어의 체력이 0이 되면 게임의 종료
   : 현재 점수와 최고 점수를 기록

2. UI 기능의 구현
- 플레이어
   : 줄어드는 체력바를 제작하여 플레이 가능 시간을 표현
- 게임
   : 게임의 실행을 알리는 Start 텍스트 출력
   : 게임 플레이 중 갱신되는 현재 점수 출력
   : 게임의 종료를 알리는 텍스트와 현재 점수 출력
   : 게임 종료 시 최고 점수를 출력

3. 추가 컨텐츠의 구현
- 추가 점수 아이템 = Gem
   : 플레이어와 충돌시 사라지며, 점수를 올려주는 Gem 아이템의 구현
   : Ground와 Top, Hurdle을 피해서 랜덤으로 생성
- 특수 능력 아이템 = Big Potion
   : Ground와 Top, Hurdle을 피해서 랜덤으로 생성
   : 일정시간동안 플레이어의 크기를 커지게 만드는 아이템의 구현
   : 아이템이 작동하는 동안 플레이어는 Hurdle과의 충돌을 무시
- 특수 능력 아이템 = Magnet Potion
   : Ground와 Top, Hurdle을 피해서 랜덤으로 생성
   : 일정시간동안 플레이어가 자성을 가지고 Gem과 Potion 아이템을 흡수
- 장애물
   : 플레이 시간이 늘어날수록 장애물이 생성되는 속도가 빨라짐

4. 애니메이션/배경 구현
- 맵
   : 맵에 에셋을 이용하여 그래픽 구현
- 플레이어
   : 플레이어의 점프와 슬라이딩 애니메이션 구현
